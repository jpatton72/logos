Two bug-fixes from beta-tester reports.

## Install (Windows)

Download `Aletheia_x64-setup.exe` (always points at the latest stable build) or the version-pinned `Aletheia_0.1.4_x64-setup.exe` below. Double-click, accept the SmartScreen warning ("More info → Run anyway"), and follow the prompts. Upgrades v0.1.0–v0.1.3 in place — your notes, bookmarks, reading position, AI chat history, and saved API keys carry over.

## Bug fixes

### Word tooltip clipped on the right edge
Clicking a Hebrew or Greek word near the right side of the window pushed the pop-up off-screen. The clamp logic was still using the v0.1.1 popup width (280px) after v0.1.2 widened it to 340px, so right-edge clicks lost the last ~60px of the panel. Now properly clamps to `window.innerWidth - 352` and also clamps top so the panel can't escape the bottom either.

### Hover-to-highlight too subtle to see
v0.1.3 introduced hover-on-Hebrew/Greek → highlight matching English in the parallel KJV verse, but the highlight was a 30% amber overlay, easy to miss against the cream reading background. Bumped to 55% opacity, full-bold weight, and added a 1px amber outline so the highlighted word stands out unmistakably even in peripheral vision. Same treatment in dark mode, tuned for the dark cream/gold palette.
