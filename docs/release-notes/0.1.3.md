Three substantial reader/study UX improvements this release.

## Install (Windows)

Download `Aletheia_x64-setup.exe` (always points at the latest stable build) or the version-pinned `Aletheia_0.1.3_x64-setup.exe` below. Double-click, accept the SmartScreen warning ("More info → Run anyway"), and follow the prompts. If you already have v0.1.0–v0.1.2 installed, this upgrades in place — your notes, bookmarks, reading position, AI conversation history, and saved API keys carry over.

WebView2 runtime required (preinstalled on Windows 11 and current Windows 10). If launch fails, install from <https://developer.microsoft.com/microsoft-edge/webview2/>.

## What changed

### Hover Hebrew/Greek → highlight English (new)
Mousing over any original-language word in a verse — in the parallel WLC/SBLGNT row, or in the Strong's sidebar — now highlights every English word in the parallel KJV verse that translates from it. Works for both 1-to-1 mappings (`ἰδοὺ` → "Behold") and 1-to-N (compound forms whose multiple English tokens share the same Strong's tag). Powered by per-verse token alignment from eBible.org's KJV2006 USFM (Public Domain), ingested into a new `english_word_alignment` table at schema v5. KJV apocrypha and non-KJV translations fall back to plain text — no regression there.

### AI conversation history (new)
Every chat in the AI panel now auto-saves to a new `ai_conversations` table after each assistant turn. The panel header gains a **History** toggle showing every past conversation as a clickable card with title, preview, message count, and "x time ago." Click a card to load it back into the active panel and continue. Per-card delete; the active conversation gets a colored border in the list. **Clear** is renamed **New chat** since saved rows persist now — to fully discard a conversation, use Delete in the history view. Schema v3 → v4.

### AI panel: auto-scroll
Sending a new message and receiving an AI response now smoothly scrolls the conversation to the bottom. Honors the OS "reduce motion" preference.

### Notes form lives in the sidebar
The note Edit / New form used to pop a centered modal that covered the reader. Now both views (and the read-only detail view) all live in the same 360px sidebar column — list, detail, and form swap in place via a Back button in the header. The reader stays interactive throughout.

### Strong's tooltip clarification labels (carried from v0.1.2)
Every item in the pop-up is explicitly labeled — `HEBREW WORD`/`GREEK WORD` caption, `Dictionary form:`, `Strong's # G2424` with hover tooltip explaining the numbering system, full `Hebrew`/`Greek` language label, `Transliteration:` / `Pronunciation:` / `Source code:`.

### Search → reader scroll + flash (carried from v0.1.2)
Clicking a search result navigates to the reader, scrolls the verse into the viewport center, and applies a 2.2-second amber flash so your eye lands on the right line.

## Linux + macOS

Build from source per [README.md](https://github.com/jpatton72/Aletheia/blob/v0.1.3/README.md). The schema migrations are forward-compatible: an existing user DB picks up the new tables empty on first launch. To populate `english_word_alignment` (the hover-to-highlight feature), run `python3 scripts/ingest_kjv_strongs.py` after upgrading.
