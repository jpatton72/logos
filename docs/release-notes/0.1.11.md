Hotfix for distorted/maxed-volume verse narration introduced in 0.1.8.

## What was broken

Piper's `--output_file -` flag writes **raw 16-bit PCM samples** to stdout, not a WAV-formatted stream. 0.1.8 took those bytes and handed them to the browser tagged as `audio/wav` — which then tried to interpret the first 44 PCM samples as a RIFF header (garbage), failed silently, and fell back to playing the rest at the wrong sample rate, wrong amplitude, and effectively maxed volume. Result: every verse came out clipped, distorted, and louder than the system mixer should have allowed.

## Fix

Synthesis now reads the voice's audio parameters from its `.onnx.json` config (sample rate, channels, bit depth) and prepends a properly-constructed 44-byte RIFF/WAVE header to Piper's raw PCM output before returning the bytes. We also detect the case where a future Piper build *does* emit a real RIFF header on its own and pass through unchanged so we don't double-wrap.

For en_US-amy-medium that's mono 16-bit 22050 Hz — what the browser was *guessing* before, but with correct framing now so the samples actually decode at the right speed and amplitude.

Three new unit tests pin the WAV header layout (RIFF/WAVE/data magic, format/channel/rate fields, chunk-size math) so a future edit can't silently regress this.

## Upgrade

Standard. The fix is entirely server-side — no need to re-download Piper or the voice model. Just install 0.1.11 and the existing `~/.local/share/aletheia/audio/` install keeps working.
