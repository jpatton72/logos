Hotfix for Linux installs where the reader was stuck on "Loading books..." indefinitely.

## What was broken (0.1.7 + 0.1.8 on Linux)

The bundled `aletheia.db` resource is what populates the book list, verses, lexicons, and word mappings on first launch. The seed routine looked for it next to the executable — which works on Windows (`C:\Program Files\Aletheia\aletheia.db` sits next to `aletheia.exe`) but **not** on Linux, where Tauri places resources at `/usr/lib/aletheia/aletheia.db` while the binary lives at `/usr/bin/aletheia`. AppImage's mount mirrors the same `usr/bin` ↔ `usr/lib/<name>` split.

Result on Linux: the seed silently skipped, the user database was created empty, and the reader's "Loading books..." placeholder never resolved.

## Fix

The seed (and the schema-upgrade backfill that has the same bug) now searches every known per-platform layout:

- `<exe-dir>/aletheia.db` — Windows installer + dev builds
- `<exe-dir>/../lib/<binary-name>/aletheia.db` — Linux .deb / AppImage
- `<exe-dir>/../share/<binary-name>/aletheia.db` — distros that route data into `/usr/share`
- `<exe-dir>/../Resources/aletheia.db` — macOS placeholder for future bundling

Every attempted candidate is logged on miss, so any future packaging quirk surfaces as an actionable diagnostic instead of a silent UI hang.

## Upgrade

### Windows
Download the new `Aletheia_x64-setup.exe` from the Releases page and reinstall over the top.

### Linux
- AppImage:
  ```bash
  wget https://github.com/jpatton72/Aletheia/releases/latest/download/Aletheia_x64.AppImage
  chmod +x Aletheia_x64.AppImage
  ./Aletheia_x64.AppImage
  ```
- Debian/Ubuntu:
  ```bash
  wget https://github.com/jpatton72/Aletheia/releases/latest/download/Aletheia_amd64.deb
  sudo dpkg -i Aletheia_amd64.deb
  ```

If you previously launched 0.1.7 or 0.1.8 on Linux, you'll have an empty-but-schema'd `~/.local/share/aletheia/data/aletheia.db` from those releases' failed seed. 0.1.9's seed now also detects that case (DB exists but has zero books) and overwrites it from the bundle on next launch. **No manual cleanup required.**
