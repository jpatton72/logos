Restores 1 Enoch 91:14, missing from 0.1.14, plus the supplement mechanism that made adding it clean.

## Fix

0.1.14 left 1 Enoch chapter 91 with a gap at verse 14 because the upstream transcription on pseudepigrapha.com is missing it. Charles's 1917 printed edition does include it — the verse reads:

> *And after that, in the ninth week, the righteous judgement shall be revealed to the whole world, and all the works of the godless shall vanish from all the earth, and the world shall be written down for destruction. And all mankind shall look to the path of uprightness.*

It's now in the bundled database. Chapter 91 is contiguous 1–19.

## Supplement mechanism

Rather than hand-edit the scraped JSON (and lose those edits if we ever re-scrape), 0.1.15 introduces [data/pseudepigrapha/supplements.json](https://github.com/jpatton72/Aletheia/blob/main/data/pseudepigrapha/supplements.json). The ingester merges this file into each book's data, **only** filling chapter+verse slots that the base scrape left empty — never overrides existing text. Re-running the scraper produces the same base JSONs and the supplement is reapplied. Future printed-vs-online discrepancies for these books get tracked in this file with provenance notes.

## Upgrade for existing 0.1.14 installs

The backfill that runs at startup now also detects when the bundled DB has more pseudepigrapha verses than the user's DB does and tops up the difference. Anyone already on 0.1.14 will get 91:14 (and any future supplement) on next launch — no manual cleanup, no DB wipe.
