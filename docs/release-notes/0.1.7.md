First Linux release. No code changes.

## Install

### Windows
Same as before — `Aletheia_x64-setup.exe` (always-latest) or the version-pinned variants below.

### Linux (new!)
Two flavors:

- **`Aletheia_x64.AppImage`** — single self-contained file, no install, no root. Download, `chmod +x`, run. Works on every modern distro.
  ```bash
  wget https://github.com/jpatton72/Aletheia/releases/latest/download/Aletheia_x64.AppImage
  chmod +x Aletheia_x64.AppImage
  ./Aletheia_x64.AppImage
  ```
- **`Aletheia_amd64.deb`** — system-wide install for Debian / Ubuntu / Mint.
  ```bash
  wget https://github.com/jpatton72/Aletheia/releases/latest/download/Aletheia_amd64.deb
  sudo dpkg -i Aletheia_amd64.deb
  ```

Both ship with the same fully populated bundled database as the Windows installer (KJV, NKJV, ESV, SBLGNT, WLC, KJV Apocrypha, 444k Hebrew + Greek word mappings, Strong's lexicons, KJV-token alignment).

The AI assistant on Linux still needs a running Secret Service provider for the OS credential vault — gnome-keyring on GNOME, KWallet on KDE, KeePassXC's secret-service integration on minimal/tiling-WM setups.

## Build & release pipeline

Releases are now produced by [`.github/workflows/release.yml`](https://github.com/jpatton72/Aletheia/blob/main/.github/workflows/release.yml) on every `git push --tags`. The workflow runs Windows + Linux jobs in parallel with shared caches for cargo, npm, and the ingest source corpora — no more local builds for cross-platform releases. See [BUILD.md](https://github.com/jpatton72/Aletheia/blob/main/BUILD.md) for the new flow.
