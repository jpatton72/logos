Diagnostic build for the v0.1.4 hover-highlight bug report. Same features as v0.1.4, plus production DevTools so the tester can capture what's happening.

## Install (Windows)

Download `Aletheia_x64-setup.exe` (always points at the latest stable build) or the version-pinned `Aletheia_0.1.5_x64-setup.exe` below. Upgrades v0.1.0–v0.1.4 in place.

## What changed

### DevTools enabled in production
Right-click anywhere in the window → **Inspect Element** (or press **Ctrl+Shift+I**) opens DevTools in installed builds. Useful for filing bug reports — testers can capture the console + DOM without running a dev build. We can drop this back to debug-only on a stable polish pass.

### Hover-highlight diagnostic
Hovering an original-language word in the reader now logs a single line to the DevTools console of the form:

```
[hover-highlight] id=H7225 matches=3 total-eng-tokens-in-chapter=234
```

If `matches=0` the highlight isn't finding anything to light up; if `total=0` the KJV verse isn't rendering as tokens. Either way the line tells me exactly which path of the bug report is the actual failure mode. Will be removed once the underlying bug is identified.
